# App Directory Structure

This directory contains the Next.js 14 App Router application structure.

## Organization

```
app/
├── (legal)/              # Route group: legal pages (URL: /privacy, /terms, /cookies)
│   ├── privacy/
│   ├── terms/
│   └── cookies/
├── api/                  # API routes
│   ├── blog/            # GET /api/blog - fetch all blog posts
│   └── contact/         # POST /api/contact - submit contact form
├── blog/                # /blog - blog listing and expanded view
├── contact/             # /contact - contact page
├── layout.tsx           # Root layout with providers, fonts, metadata
├── page.tsx             # / - home page
├── manifest.ts          # PWA manifest
├── robots.ts            # SEO robots configuration
└── sitemap.ts           # Dynamic sitemap generation
```

## Route Groups

Route groups (folders with parentheses like `(legal)`) are used to organize pages without affecting URL structure:
- `/privacy`, `/terms`, `/cookies` URLs remain unchanged
- Improves code organization and maintainability

## Key Files

- **layout.tsx**: Root shell with theme provider, fonts, security headers, analytics
- **page.tsx**: Home page with hero, projects, technologies, CTA
- **robots.ts**: Crawler rules and sitemap location (generates `/robots.txt`)
- **sitemap.ts**: Dynamic XML sitemap with all pages and blog posts
- **manifest.ts**: PWA manifest (optional, auto-generated by Next.js)

## Adding New Routes

To add a new route:
1. Create a folder: `app/new-route/page.tsx`
2. Export a default React component
3. Add metadata if SEO is important
4. Update `sitemap.ts` if it should be indexed

Example:
```typescript
// app/portfolio/page.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "Portfolio",
  description: "My work and projects"
};

export default function PortfolioPage() {
  return <h1>Portfolio</h1>;
}
```

## API Routes

API routes in `api/` are serverless functions:
- `POST /api/contact` - Contact form submission with rate limiting, validation, security
- `GET /api/blog` - Fetches all blog posts from `content/blog/` with caching

## Best Practices

- ✅ Use route groups to organize pages logically
- ✅ Add metadata to all pages for SEO
- ✅ Use dynamic content generation (sitemap, robots)
- ✅ Implement proper error handling and validation
- ✅ Use environment variables for sensitive data
